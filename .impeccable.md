# Impeccable Design Context — LARP Nexus

## Design Context

### Users

**GM（遊戲主持人）**
- 主要使用 PC 桌面瀏覽器
- 在遊戲前進行大量設定工作（角色、道具、技能、劇本）
- 遊戲進行中需要快速掌握全局資訊與即時推播
- 情境：可能在燈光昏暗的活動場地使用，需要清晰可讀的介面
- 任務：管理劇本 → 編輯角色 → 監控遊戲進行

**玩家**
- 主要使用手機（單手操作為主）
- 遊戲進行中需要快速查詢角色資訊、使用道具/技能
- 對抗流程有時間壓力，關鍵操作必須直覺
- 情境：站立或走動中使用，光線條件不穩定
- 任務：瀏覽角色卡 → 使用道具/技能 → 回應對抗 → 查看通知

### Brand Personality

**三個核心詞**：優雅（Elegant）・精緻（Refined）・有質感（Premium）

**語調**：沉穩而帶有儀式感。介面本身是遊戲沉浸體驗的一部分，而非只是工具。
**情感目標**：玩家打開角色卡時感到「這是我的角色」的代入感；GM 管理介面帶來「掌控全局」的沉穩自信。

### Aesthetic Direction

**視覺風格**：神秘奇幻（Mystical Fantasy）+ 優雅精緻

**深色模式**（玩家主要使用）
- 背景：深午夜藍（如 oklch(0.12 0.02 265)）而非純黑，保留空間感
- 主強調色：金/琥珀（如 oklch(0.75 0.15 75)），用於主要行動按鈕、數值高亮
- 次強調色：深紫/靛（如 oklch(0.35 0.08 280)），用於卡片背景、分隔層次
- 邊框：帶微弱光暈的半透明線（如 oklch(1 0 0 / 12%)）

**淺色模式**（GM 工作時使用）
- 背景：暖米白（如 oklch(0.98 0.005 80)）而非純白
- 主強調色：同一金/琥珀，維持品牌一致性
- 卡片：帶輕微暖調陰影，非冷灰色陰影

**字體**：繼續使用 Geist，標題搭配適度字重對比（font-semibold/bold），不引入裝飾性字體以保持可讀性

**動畫**：利用 Framer Motion 製造有質感的進場/轉場（fade + slide，200–350ms），避免過度動畫干擾操作

**不要**：
- 過於沉重的哥德風（粗黑邊框、骷髏等裝飾）
- 電玩 HUD 科技感（霓虹光暈、電子感字體）
- 純白/純黑背景（缺乏奇幻氛圍）
- Bootstrap 風格的通用感

### Design Principles

1. **沉浸優先（Immersive by Default）**
   每個元件的視覺設計都應強化 LARP 的奇幻氛圍，而非只是功能性的表單與按鈕。

2. **關鍵操作一步到位（One-Thumb Reachability）**
   玩家端所有主要操作（使用道具、使用技能、回應對抗）必須在手機上單手可達，放置於畫面下半部。

3. **資訊層次清晰（Scannable Hierarchy）**
   GM 端資訊密度高，必須透過尺寸、顏色、間距建立清楚的視覺層次，讓眼睛能快速定位重點。

4. **深淺模式同等精緻（Theme Parity）**
   深色模式（玩家）與淺色模式（GM）都必須是精心設計的，而非其中一個是「另一個的反色」。

5. **環境感知的視覺提示（Context-Aware States）**
   Baseline / Runtime 環境切換、對抗進行中、道具耗盡等狀態，都需要明確但不干擾的視覺提示，讓 GM 與玩家隨時了解當前狀態。

## Tech Stack Constraints

- **框架**：Next.js 16 + React 19 + TypeScript（strict）
- **樣式**：Tailwind CSS 4（oklch 色彩空間）
- **組件庫**：shadcn/ui（Radix UI primitives）—— 可自訂 token，不可替換底層 primitive
- **動畫**：Framer Motion（已安裝，優先使用）
- **圖示**：Lucide React
- **Toast**：Sonner
- **模式支援**：next-themes（深淺模式）
- **語言**：繁體中文介面（lang="zh-TW"）
- **RWD 優先順序**：玩家端手機優先，GM 端桌面優先，兩者均需支援

## Screen Inventory

### GM 端
- 登入畫面（Magic Link 驗證）
- 主畫面（入口 hub）
- 劇本管理（建立/選擇劇本）
- 管理劇本（Baseline/Runtime 切換、劇本資訊、角色列表）
- 編輯角色（基本資訊、角色數值、任務管理、道具管理、技能管理）

### 玩家端
- 公開頁面（世界觀/前導故事）
- 角色卡解鎖畫面
- 角色卡預設模式（唯讀瀏覽）
- 角色卡完整模式（可互動、對抗流程）
- 通知面板
