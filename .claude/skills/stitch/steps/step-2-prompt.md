# Step 2：撰寫 Stitch Prompt

## 必填欄位

使用純文字清單格式（**不使用表格**，方便用戶直接複製貼上到 Stitch）：

```
畫面名稱：[頁面標題]
主要功能：[這個畫面的目的]
互動元素：[列出所有按鈕、輸入框、切換開關]
操作邏輯：[每個互動的行為描述]
模式差異：[若有多種模式，說明差異；無則寫「無」]
錯誤狀態：[所有可能的錯誤顯示]
RWD 要求：[目標裝置，例如：手機優先（320px–768px）]
主題：[深色／淺色／兩者]

設計方向：[風格描述]

頁面結構（由上而下）：
1. [區塊名稱]
   - [細節描述]
2. [區塊名稱]
   - [細節描述]
...

視覺 Token：
- [token 名稱]：[用途]
```

> **重要**：互動元素和操作邏輯缺一不可，否則 Stitch 會省略按鈕或狀態。
