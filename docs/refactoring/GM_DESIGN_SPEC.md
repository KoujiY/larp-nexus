# GM 端全面重設計 — 設計規格

**版本**：2.0（取代原 v1.0 2026-03-24 版本，舊版歸檔於 `docs/archive/gm-design-spec-v1.md`）
**日期**：2026-04-01
**分支**：`refactor/project_enhancement`（Phase D-5b-2）
**設計參考**：`docs/specs/DESIGN.md`、`.impeccable.md`

---

## 1. 設計決策摘要

以下為 brainstorming 階段確認的設計方向：

- **導航結構**：砍掉 Dashboard，劇本列表即首頁（頂部帶歡迎區）
- **Runtime 介面策略**：共用框架 + Runtime 控制台 Tab（Baseline 時隱藏）
- **角色編輯 Tab 拆分**：從 5 個拆為 7 個（基本設定、背景故事、隱藏資訊、數值、任務、道具、技能）
- **儲存策略**：全局 Sticky Save Bar，跨 Tab dirty state 保留，一鍵全部儲存
- **道具/技能編輯**：從單頁 Dialog 改為 4 步驟 Stepper Wizard
- **Log 系統**：新功能，通知 + 操作紀錄，可按角色/事件類型篩選
- **個人設定**：最小化，不過度投入

## 2. 路由結構

```
/games                                    → 首頁（劇本列表 + 歡迎區）
/games/[gameId]                           → 劇本管理頁
/games/[gameId]/characters/[characterId]  → 角色編輯頁
/profile                                  → 個人設定
```

側邊欄導航項：劇本管理（主入口）、個人設定、登出。

## 3. 共用設計規範

### 3.1 環境色彩系統

- Baseline（設定模式）：`bg-env-baseline` 板岩藍橫幅，`text-env-baseline` 標籤
- Runtime（遊戲進行中）：`bg-env-runtime` 琥珀金橫幅，`text-env-runtime` 標籤
- 環境橫幅固定在內容區頂部，所有子頁面可見

### 3.2 儲存系統

#### 三層儲存架構

| 層級 | 適用範圍 | 「儲存」的意思 | 寫入目標 |
|------|---------|---------------|---------|
| Dialog 層 | 道具/技能 Wizard、隱藏資訊編輯 Dialog | 「確認這筆編輯」 | 父 Tab 的記憶體狀態 |
| Tab 層 | 角色編輯頁的 7 個 Tab | Sticky Save Bar 統一處理 | 伺服器 |
| 頁面層 | 劇本管理頁的劇本資訊 Tab | 獨立 SaveButton | 伺服器 |

流程範例：GM 開啟道具 Wizard → 編輯完按「儲存道具」→ 道具資料寫入道具 Tab 的記憶體陣列 → 道具 Tab 標記 dirty → Sticky Save Bar 出現 → GM 按「全部儲存」→ 所有 dirty Tab 一次送出至伺服器。

#### Sticky Save Bar（角色編輯頁專用）

- 底部固定浮動列，有 dirty state 時自動出現
- 顯示摘要：「N 個分頁有未儲存的變更（Tab 名稱列表）」
- 詳細統計：包含新增、修改、刪除的項目數量（例如「道具：新增 2、修改 1、刪除 1」）
- 「全部儲存」一鍵送出、「捨棄變更」回復所有 Tab（含復原已標記刪除的項目）
- 切 Tab 不遺失、不跳警告
- 離開頁面：beforeunload + 自訂 AlertDialog

#### 列表項目的未儲存狀態標記

所有列表型資料（道具、技能、任務、隱藏資訊、數值、人物關係）在 Tab 內需要顯示個別項目的 dirty 狀態：

| 狀態 | 視覺提示 | 觸發情境 |
|------|---------|---------|
| 新增（未儲存） | 卡片左邊框琥珀金 + 「新增」小 badge（warning 色） | Wizard / Dialog 建立完回到列表 |
| 已編輯（未儲存） | 卡片左邊框琥珀金 + 「已修改」小 badge（warning 色） | Wizard / Dialog 編輯完回到列表 |
| 已儲存（正常） | 無特殊標記 | Sticky Save Bar 儲存成功後清除所有標記 |
| 刪除標記（未儲存） | 卡片半透明（opacity-50）+ 刪除線 + 「已標記刪除」badge（destructive 色）| 按刪除按鈕後，標記為待刪除 |

刪除行為：按下刪除按鈕**不會立即刪除**，而是標記為待刪除。真正的刪除在「全部儲存」時執行。「捨棄變更」可復原所有標記刪除的項目。

#### 劇本管理頁儲存

劇本管理頁的劇本資訊 Tab 使用 Tab 內部的獨立 SaveButton（非 Sticky Save Bar），因為只有一個可編輯 Tab，不存在跨 Tab 遺失問題。

### 3.3 表單設計通則

- 欄位層級錯誤：欄位下方 `text-destructive text-sm`
- 表單層級錯誤：Toast `toast.error()`
- 成功：Toast `toast.success()`
- Dialog 最大寬度：`max-w-3xl`（768px），行動版 `max-w-[95vw]`
- Dialog 內部滾動：`overflow-y-auto max-h-[80vh]`

### 3.4 空狀態規範

所有列表的空狀態：`bg-muted/30 rounded-xl py-16`，icon `h-10 w-10 text-muted-foreground/40`，無虛線邊框。

### 3.5 側邊欄導航

**桌面版（`lg:` 以上）**：固定展開側邊欄
- Logo + 品牌名（Drama icon）
- 導航項：劇本管理（主入口）、個人設定
- 底部固定：登出
- 目前頁面：`bg-primary/10 text-primary font-medium`
- 非目前頁面：`text-foreground hover:bg-muted`
- Icon 尺寸：`h-4 w-4`

**行動版**：`MobileHeader`（已實作）
- Header 高度 `h-14`，Logo + 漢堡選單
- Sheet 內容與桌面版相同

### 3.6 動畫規範

| 情境 | 動畫 | 時長 |
|------|------|------|
| 頁面切換 | fade-in | 200ms |
| Tab 切換 | fade | 150ms |
| 卡片 hover | lift (`translateY(-2px)`) | 150ms |
| Dialog 開啟 | scale (0.96→1) + fade | 200ms |
| 儲存成功 | Save button brief pulse | 300ms |
| 環境橫幅出現 | slide-down | 250ms |

### 3.7 無障礙（a11y）基線

- 所有表單欄位有 `<label htmlFor>`，不用 placeholder 代替 label
- 確認 Dialog（刪除、切換環境）需有明確的「破壞性操作」視覺警示
- 麵包屑使用 `<nav aria-label="breadcrumb">` + `<ol>` 結構
- 表格和網格有適當的 `role` 和 `aria-label`

### 4.4 劇本管理頁 — Runtime 狀態（控制台）

**路由**：同 `/games/[gameId]`，遊戲進行中
**情感目標**：遊戲中的指揮中心——監控、溝通、一切盡在掌握。

**規格**：
- 環境橫幅：Runtime 樣式（琥珀金底，「遊戲進行中」標籤 + 劇本名稱）
- Tab 列：**控制台**（預設選中）· 劇本資訊 · 角色列表
- 控制台 Tab 僅在 Runtime 時出現

控制台內容分為四個模組：

**狀態總覽**：
- 角色卡片橫列或網格
- 每張卡片：頭像/初始字、角色名稱、關鍵數值（有 maxValue 的 stat 顯示為進度條）
- 數值 ≤25% 時標紅
- 點擊角色卡跳轉角色編輯頁
- 角色 >6 時可捲動

**事件 Log**：
- 全局事件流，即時更新（WebSocket）
- 篩選器：角色下拉 + 事件類型 checkbox
- 每條 Log：時間戳、角色名稱、事件摘要
- 可展開查看詳細數值
- 大量 Log 時 lazy loading

**快速廣播**：
- 模式切換：全體廣播 / 指定角色（Toggle）
- 指定角色模式下顯示角色下拉選單
- 標題輸入（必填）+ 訊息輸入（選填）
- 發送按鈕
- 發送後紀錄出現在 Log 中

**遊戲控制**：
- 「結束遊戲」按鈕（destructive 樣式）
- 觸發確認 Dialog

**Stitch prompt**：

```

使用者情境：
GM 正在主持一場 LARP 遊戲，可能在活動現場，需要即時監控所有角色狀態、發送廣播訊息給玩家、查看遊戲事件紀錄。這是遊戲進行中 GM 最頻繁使用的畫面。

- 淺色主題，暖米白背景

1. 環境橫幅
- 橫跨內容區頂部的琥珀金色橫條
```

**規格**：

4 步驟 Stepper：

**步驟 1 — 基本資訊**：
- 名稱（必填）
- 描述（Textarea）
- 道具專屬：類型（消耗品/裝備）、數量、可轉移 Switch
- 技能專屬：icon URL
- 標籤（combat / stealth checkbox）

**步驟 2 — 檢定系統**：
- 檢定類型選擇：無檢定 / 對抗檢定 / 隨機檢定 / 隨機對抗檢定
- 根據類型顯示對應設定：
  - 對抗檢定：關聯數值、防守方可用道具/技能、平手處理
  - 隨機檢定：關聯數值、成功門檻、成功/失敗效果指定
  - 隨機對抗檢定：結合以上兩者

**步驟 3 — 使用限制**：
- 使用次數限制（數字，0 = 無限制）
- 冷卻時間（分鐘）

**步驟 4 — 效果設計**：
- 效果列表：master-detail 佈局（左側列表 + 右側詳情）
- 效果類型：數值變更、自訂效果、奪取道具、偷竊道具、揭露任務（技能專屬）、完成任務（技能專屬）
