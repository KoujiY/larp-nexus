# GM 端設計規範

**版本**：1.0
**建立日期**：2026-03-24
**設計參考**：`.impeccable.md`（品牌個性、色彩系統、技術約束）
**對應計畫**：`PHASE_D_PLAN.md` D-4 GM 側頁面設計規範

---

## 總體設計方向

GM 端的核心情感目標：**「掌控全局」**。

GM 是這場遊戲的導演。介面應讓他們感到沉穩自信——資訊清晰可讀、操作有確定感、永遠知道自己在哪個環境做什麼事。

### 佈局原則

- **桌面優先**（1280px 基準），手機為次要（MobileHeader + Sheet 已實作）
- 側邊欄導航（`lg:` 以上固定展開）
- 內容區最大寬度：`max-w-6xl`（1152px），大量資訊的表單頁可到 `max-w-5xl`
- 資訊密度高時，用間距與色彩分層，而非加更多邊框

### 淺色模式為主

GM 工作環境以白天為主，淺色模式（暖米白背景）提供更好的長時間閱讀體驗。設計規範以淺色模式為基準，深色模式自動跟隨 token 系統。

### 環境色彩系統（C-07 修復核心）

這是 GM 端最重要的設計決策。**Baseline 與 Runtime 必須在視覺上立即可辨認。**

| 環境 | Token | 使用位置 |
|------|-------|---------|
| Baseline（設定中）| `bg-env-baseline` / `text-env-baseline` | 頁面標題 badge、頂部橫幅 |
| Runtime（遊戲中）| `bg-env-runtime` / `text-env-runtime` | 頁面標題 badge、頂部橫幅、警示 banner |

**環境橫幅規範**（出現於劇本管理頁和角色編輯頁的頂部）：

```
┌─────────────────────────────────────────────────────┐
│  ● 遊戲進行中（Runtime）          [劇本名稱]          │  ← bg-env-runtime/15, border-b border-env-runtime/30
└─────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────┐
│  ○ 設定模式（Baseline）           [劇本名稱]          │  ← bg-muted/50, border-b border-border
└─────────────────────────────────────────────────────┘
```

橫幅始終固定在內容區頂部（在 MobileHeader 下方），確保 GM 在任何子頁面都能感知環境。

## 畫面規範

### 1. 側邊欄導航（Navigation）

**桌面版（`lg:` 以上）**：

```
┌──────────────────┐
│  ◆ LARP Nexus    │  ← Logo + 品牌名，帶 Drama icon
├──────────────────┤
│  [主畫面]         │
│  [劇本管理]       │  ← 主要導航項，icon + 文字
│  [個人設定]       │
├──────────────────┤
│  [登出]           │  ← 底部固定
└──────────────────┘
```

**行動版**：已實作 `MobileHeader`（頂部 header + Sheet 側拉式選單），此為完整規範：
- Header 高度 `h-14`，顯示 Logo + 漢堡選單按鈕
- Sheet 內容與桌面版側邊欄相同

**導航項目視覺**：
- 目前頁面：`bg-primary/10 text-primary font-medium`
- 非目前頁面：`text-foreground hover:bg-muted`
- Icon 尺寸：`h-4 w-4`，與文字垂直居中

### 2. 主畫面 Dashboard（`/dashboard`）

**情感目標**：快速找到入口，不需思考，充滿掌控感。

**現況問題**：陽春的 Quick Actions + 快速指南，毫無品牌感。

#### 佈局結構

```
┌─────────────────────────────────────────────────────┐
│  歡迎回來，[GM 名稱]                                  │  ← text-2xl，帶 GM 名字個人化
│  [最後登入時間，text-sm text-muted-foreground]        │
├─────────────────────────────────────────────────────┤
│  ┌──────────────────┐  ┌──────────────────┐         │
│  │  劇本管理         │  │  個人設定         │         │  ← 2 欄 Card，hover 有輕微 lift 效果
│  │  [BookOpen icon] │  │  [Settings icon] │         │
│  │  進入管理劇本 →   │  │  查看帳號資料 →   │         │
│  └──────────────────┘  └──────────────────┘         │
├─────────────────────────────────────────────────────┤
│  ▸ 快速流程說明（可摺疊，預設展開）                    │
│    1. 建立劇本  2. 新增角色  3. 設定道具/技能           │
│    4. 開始遊戲  5. 結束遊戲                           │
└─────────────────────────────────────────────────────┘
```

#### 設計細節

- **入口卡片**：帶 `hover:shadow-md hover:-translate-y-0.5 transition-all` 的 lift 效果
- **移除「角色卡」入口**：角色卡需先選劇本才能進入，直接放 dashboard 會造成困惑
- **空狀態版本**：若 GM 尚未建立任何劇本，在 dashboard 顯示「開始你的第一個劇本」CTA

### 4. 劇本管理頁（`/games/[gameId]`）

**情感目標**：GM 的指揮中心。進入時要立刻感知當前環境（Baseline or Runtime），並快速找到需要的操作。

**這是最複雜的 GM 頁面，C-07 的主要修復位置。**

#### 佈局結構

```
┌─────────────────────────────────────────────────────┐
│  [環境橫幅]  ← 見上方「環境色彩系統」規範              │
├─────────────────────────────────────────────────────┤
│  劇本列表 / [劇本名稱]  ← 麵包屑                     │
│  [劇本名稱]（text-3xl bold）  [環境 badge]            │
├─────────────────────────────────────────────────────┤
│  [操作區]                                            │
│  左：GameCodeSection（顯示/複製 Game Code）           │
│  右：GameLifecycleControls（開始/停止遊戲）            │
│       + QR Code、預覽、刪除 按鈕群                    │
├─────────────────────────────────────────────────────┤
│  [GameEditTabs]                                      │
│  ┌──────────┬──────────┬──────────┐                 │
│  │ 劇本資訊 │  廣播系統 │  角色列表 │                 │
│  └──────────┴──────────┴──────────┘                 │
│  [Tab 內容]                                          │
└─────────────────────────────────────────────────────┘
```

#### GameLifecycleControls 設計

這是最關鍵的操作按鈕，必須有確定感且不可誤觸。

**待機中狀態（Baseline）**：
```
[▷ 開始遊戲]  ← bg-env-runtime text-env-runtime-foreground，帶確認 Dialog
```

**進行中狀態（Runtime）**：
```
[■ 結束遊戲]  ← variant="destructive"，帶確認 Dialog（說明後果：Runtime 資料將被刪除）
```

兩個按鈕都必須觸發確認 Dialog，**不允許單擊直接執行**。

#### CharacterEditTabs 設計

**標籤列**（桌面版水平，行動版可滾動）：
- Tab 文字 + icon
- 有未保存變更的 Tab：名稱後加 `●`（`text-warning`）

**未保存變更提示（取代 `window.confirm()`）**：
切換到有未保存變更的 Tab 時，改用 `AlertDialog`（shadcn/ui）：
```

[留在此頁]  [不儲存，離開]
```
這比 `window.confirm()` 更符合設計語言，且可自訂樣式。

## 表單設計通則

GM 端大量使用表單，以下為通用規範：

### 儲存狀態

每個子表單（StatsEditForm、TasksEditForm 等）使用 `SaveButton` 元件，狀態如下：

| 狀態 | 顯示 | 樣式 |
|------|------|------|
| 乾淨（無變更）| 「已儲存」+ Check icon | `variant="outline"` disabled |
| 有變更 | 「儲存變更」| `variant="default"` |
| 儲存中 | Loader2 旋轉 + 「儲存中...」| `variant="default"` disabled |
| 儲存失敗 | 「儲存失敗，重試」| `variant="destructive"` |

### 錯誤顯示

- **欄位層級錯誤**：欄位下方 `text-destructive text-sm`，不使用 alert
- **表單層級錯誤**：Toast（Sonner）`toast.error()`
- **成功**：Toast `toast.success()`

### Dialog 規範

GM 端 Dialog（道具編輯、技能編輯）：
- 最大寬度：`max-w-5xl`（1024px）
- 行動版：`max-w-[95vw]`
- 內容過多時：Dialog 內部滾動（`overflow-y-auto max-h-[80vh]`），不允許 Dialog 本身超出視窗高度

## 動畫規範

| 情境 | 動畫 | 時長 |
|------|------|------|
| 頁面切換 | fade-in | 200ms |
| Tab 切換 | fade | 150ms |
| 卡片 hover | lift (`translateY(-2px)`) | 150ms |
| Dialog 開啟 | scale (0.96→1) + fade | 200ms |
| 儲存成功 | Save button brief pulse | 300ms |
| 環境橫幅出現 | slide-down | 250ms |

## 無障礙（a11y）基線

與玩家端相同基線，GM 端額外要求：

- 所有表單欄位有 `<label htmlFor>`，不用 placeholder 代替 label
- 確認 Dialog（刪除、切換環境）需有明確的「破壞性操作」視覺警示
- 麵包屑使用 `<nav aria-label="breadcrumb">` + `<ol>` 結構
- 表格和網格有適當的 `role` 和 `aria-label`

---

## Stitch Prompt 要點

生成 GM 端 Stitch prompt 時，需包含：

1. **使用者情境**：「坐在桌前的遊戲主持人，可能在活動現場，需要快速操作不出錯」
2. **情感目標**：對應畫面的「掌控全局」目標
3. **淺色模式**：暖米白背景（`oklch(0.98 0.005 80)`）+ 琥珀金主色
4. **環境區分**：Baseline（板岩藍）vs Runtime（琥珀金）必須視覺上一眼可辨認
5. **禁止項目**：無漸層紫色、無 Bootstrap 感、無電玩 HUD 風格
6. **重點畫面**：建議從「劇本管理頁（Runtime 狀態）」開始，這是最複雜且 C-07 的修復核心
