# LARP Nexus 使用指南

歡迎使用 LARP Nexus！這是一套專為**實境角色扮演遊戲（LARP）**設計的數位角色卡管理系統。

傳統 LARP 使用紙本角色卡，GM 需要手動追蹤數值變化、物品流轉、技能使用。LARP Nexus 把這些通通數位化——GM 可以即時推送劇情、遠端調整角色狀態；玩家用手機就能查看角色卡、使用技能和物品，所有操作即時同步。

> **閱讀建議**：GM 請通讀全文；玩家可以直接跳到[第二部分：玩家指南](#玩家指南)。

---

# 第一部分：GM（遊戲主持人）指南

## 1. 登入系統

LARP Nexus 使用 **Magic Link** 驗證，不需要密碼：

1. 前往首頁，點擊「GM Login」
2. 輸入你的 Email
3. 去信箱收取驗證信，點擊信中連結
4. 登入完成，自動進入 Dashboard

> Magic Link 有時效性，如果過期請重新申請。

---

## 2. 劇本管理

登入後，從 Dashboard 進入劇本管理。

### 2.1 新增劇本

1. 點擊「新增劇本」
2. 填寫劇本名稱與描述
3. 儲存後，系統會自動生成一組 **6 位英數字遊戲代碼**（如 `A3KX9P`）

> 建議每場遊戲使用不同的遊戲代碼。雖然系統允許重複，但不同劇本使用相同代碼可能導致玩家解鎖到錯誤的角色。

### 2.2 劇本頁面總覽

劇本管理頁面依遊戲狀態有不同的分頁：

**準備階段：**

| 分頁 | 功能 |
|------|------|
| **劇本資訊** | 劇本名稱、描述、遊戲代碼、世界觀內容編輯 |
| **預設事件** | 預先設計遊戲中可一鍵觸發的多動作事件（詳見[第 5 章](#5-預設事件)） |
| **角色列表** | 建立與管理所有角色（詳見[第 3 章](#3-設計角色)） |
| **角色匯入** | AI 自動解析角色資料並建立角色（詳見[第 4 章](#4-ai-角色匯入)） |

**遊戲進行中：**

| 分頁 | 功能 |
|------|------|
| **控制台** | 即時監控角色狀態、廣播訊息、觸發事件（詳見[6.4 控制台](#64-遊戲進行中控制台)） |
| **劇本資訊** | 同上（遊戲中可繼續編輯） |
| **預設事件** | 同上（遊戲中可繼續新增或編輯） |
| **角色列表** | 查看角色即時狀態 |

### 2.3 世界觀編輯

在「劇本資訊」分頁中編輯世界觀，使用**區塊式編輯器**：
- **標題區塊**：章節標題
- **內文區塊**：段落內容

你可以自由排列區塊順序，建構世界觀的敘事結構。每個劇本有一個公開的世界觀頁面，不需要 PIN 即可瀏覽。你可以把這個連結提前分享給玩家，讓他們在遊戲開始前熟悉世界觀。

### 2.4 QR Code 與連結分享

系統可以為劇本世界觀頁面和每張角色卡生成 QR Code，方便分享連結：

- **世界觀頁面 QR Code**：在「劇本資訊」分頁中生成，可下載圖片或直接複製連結
- **角色卡 QR Code**：在「角色列表」中，每張角色卡都有獨立的 QR Code 按鈕，可下載圖片

## 3. 設計角色

在劇本的「角色列表」分頁中，點擊「新增角色」。角色編輯頁面分為兩組共七個分頁：

### 分頁總覽

**敘事群組：**

| 分頁 | 內容 |
|------|------|
| **基本設定** | 名稱、描述、角色圖片、PIN 碼 |
| **背景故事** | 角色的公開背景，使用區塊式編輯器 |
| **隱藏資訊** | 角色不知道的秘密，可設定自動揭露條件 |

**機制群組：**

| 分頁 | 內容 |
|------|------|
| **數值** | 角色能力值（體力、魔力等） |
| **任務** | GM 指派的目標與隱藏任務 |
| **物品** | 角色持有的物品 |
| **技能** | 角色可使用的能力 |

> 各分頁有獨立的**未儲存標記**，修改任何分頁後，底部會出現**儲存列**，可一次儲存所有分頁的變更。

### 3.1 基本設定

- **名稱**、**描述**、**角色圖片**
- **PIN 碼**：4 位數字密碼，用來保護角色卡。開啟 PIN 鎖後，玩家必須輸入正確 PIN 才能查看

> 建議提前幾天將 PIN 碼和角色卡連結發給玩家，讓他們用預覽模式提前熟悉自己的角色和世界觀。遊戲代碼則在遊戲當天再公布。

### 3.2 背景故事

角色的公開背景資訊，使用**區塊式編輯器**：
- **標題區塊**：用於分段標題（例如「童年」、「轉折」）
- **內文區塊**：段落內容

區塊可自由新增、刪除、排序，建構出有層次的角色背景敘事。

> 注意：背景故事是「對該角色公開」，不代表其他玩家也會看到。每張角色卡都是專屬的，背景中可能包含不希望其他玩家得知的內容。

### 3.3 隱藏資訊

角色對自己不清楚的部分。這些資訊在遊戲開始時全部處於隱藏狀態，隨著遊戲推進可能會逐漸揭露：
- 每條隱藏資訊有**標題**和**內容**（內容支援多段落）
- 可設定**自動揭露條件**，讓隱藏資訊在特定情境下自動解鎖（詳見[自動揭露系統](#自動揭露系統)）
- GM 也可以在遊戲進行中手動揭露

### 3.4 數值

角色的能力值，例如體力、魔力、智力等：
- 每項數值有**名稱**、**當前值**、**最大值**
- 技能和物品的效果會修改這些數值

### 3.5 任務管理

GM 指派給角色的任務與目標：
- 每個任務有**標題**和**描述**
- 任務有四種狀態：**待處理**、**進行中**、**已完成**、**失敗**，GM 可隨時調整
- 任務可以設為**隱藏目標**，搭配自動揭露條件，在適當時機才顯示給玩家
- 隱藏目標在揭露後會以特殊樣式標示，提醒玩家這是新發現的任務
- 部分技能可透過效果自動將任務標記為完成

### 3.6 物品管理

角色持有的物品。每個物品透過**四步驟精靈**編輯：

1. **基本資訊**：名稱、描述、圖片、類型、數量、可轉移設定、使用上限、冷卻時間
2. **效果設定**（可多個，僅消耗品/道具可設定）：
   - **數值變化**：修改目標角色的數值（如扣血、補魔）
   - **偷取物品**：從目標角色偷走一件物品，移到自己身上
   - **移除物品**：從目標角色身上移除一件物品（物品會消失，不會轉移給使用者）
   - **自訂效果**：GM 自行定義的效果描述，由 GM 口頭裁定
3. **檢定方式**（僅消耗品/道具可設定）：
   - **無檢定**：直接生效
   - **隨機檢定**：系統擲骰決定成敗
   - **對抗檢定**：與目標角色比較數值，高者獲勝
   - **隨機對抗**：雙方各擲一次骰，比大小決勝負
4. **標籤與預覽**：
   - **戰鬥**：帶有此標籤的物品可以用來回應戰鬥對抗
   - **隱匿**：帶有此標籤的物品在使用時會隱藏使用者身份

#### 物品類型

| 類型 | 說明 | 使用後 |
|------|------|--------|
| **消耗品** | 一次性物品 | 每次使用扣減數量，數量歸零後無法再使用（物品仍保留在清單中） |
| **道具** | 可重複使用的物品 | 不扣減數量，可搭配使用上限和冷卻時間控制使用頻率 |
| **裝備** | 被動加成物品 | 不能主動「使用」，只能穿戴/卸下 |

#### 裝備類型

裝備不設定效果和檢定，改為設定**數值加成**：
- 穿戴時數值加成即時生效（如 +5 體力、-1 魔力上限），卸下後恢復
- 裝備被轉移給其他角色時，會自動卸下並還原加成
- 裝備不會出現在對抗回應的選項中

### 3.7 技能管理

角色可使用的能力。同樣透過**四步驟精靈**編輯：

1. **基本資訊**：名稱、描述、圖片、使用次數（0 為無限）、冷卻時間
2. **效果設定**：與消耗品/道具相同（數值變化、偷取物品、移除物品、自訂效果），另有技能專屬效果：
   - **揭露任務**：揭露目標角色的指定隱藏任務
   - **完成任務**：將目標角色的指定任務標記為完成
3. **檢定方式**：與消耗品/道具相同，對抗時可額外設定：
   - 關聯數值（用哪項數值進行對抗）
   - 對手可使用的最大物品/技能數（設為 0 則防守方無法使用物品或技能應對）
   - 平手處理規則
4. **標籤與預覽**：與物品相同。帶有「戰鬥」標籤的技能發起對抗時，防守方只能使用同樣帶有「戰鬥」標籤的物品或技能來應對

### 自動揭露系統

隱藏資訊和隱藏任務可以設定**自動揭露條件**，讓系統在條件滿足時自動解鎖：

| 條件類型 | 觸發時機 | 適用對象 | 範例 |
|----------|----------|----------|------|
| 看過指定物品 | 透過展示或其他方式看到某些物品後觸發 | 隱藏資訊、隱藏任務 | 「看過神秘信件後揭露真實身份」 |
| 獲得指定物品 | 透過偷取、轉移等方式取得某些物品後觸發 | 隱藏資訊、隱藏任務 | 「拿到鑰匙後解鎖密室任務」 |
| 特定隱藏資訊已揭露 | 角色的其他隱藏資訊被揭露後連鎖觸發 | **僅隱藏任務** | 「揭露真實身份後，連鎖揭露真正的目的」 |

條件可以設為**全部滿足**或**任一滿足**。

> 注意：「特定隱藏資訊已揭露」條件只能用於隱藏任務，無法設定在隱藏資訊上。如需隱藏資訊之間的連鎖揭露，請透過隱藏任務間接觸發，或由 GM 手動揭露。

## 4. AI 角色匯入

如果你已經有角色的文字設定（例如從文件或其他工具匯出的角色描述），可以透過 AI 自動解析並建立角色，省去手動輸入的時間。

### 4.1 前置設定：AI 服務

首次使用前，需先在**個人設定頁**設定 AI 服務：

1. 從左側導航進入「個人設定」
2. 在 AI 設定區塊選擇 Provider（OpenAI / Google Gemini / 自訂）
3. 輸入 API Key 和模型名稱（選擇 Provider 後會自動帶入預設值）
4. 點擊「儲存」，再點擊「驗證連線」確認設定正確

> API Key 以加密方式儲存，系統不保留明文。

### 4.2 匯入流程

1. 在劇本的「角色匯入」分頁中，選擇輸入方式：
   - **貼上文字**：直接貼上角色設定文字（最多 50,000 字）
   - **上傳 .docx**：上傳 Word 文件（最大 5MB）

2. 設定匯入選項：

   | 選項 | 說明 |
   |------|------|
   | **包含隱藏資訊** | 啟用後會解析秘密與隱藏任務 |
   | **允許 AI 補足** | 原文中缺少的欄位由 AI 根據角色形象補足 |
   | **自訂提示** | 額外指示（如「性格請拆成多個特質條列」，最多 500 字） |

3. 點擊「開始解析」，等待 AI 處理

4. 在預覽頁面檢視解析結果，可微調基本欄位（名稱、描述、標語）

5. 確認後系統自動建立角色，並導航至角色編輯頁繼續完善設定

> AI 匯入使用「段落索引法」——AI 只標記段落編號，程式碼從原文直接複製，確保匯入內容與原文完全一致，不會被 AI 改寫。

## 5. 預設事件

GM 可以在「預設事件」分頁中預先設計好事件，在遊戲進行中一鍵觸發。這對於需要同時影響多個角色或執行多步驟操作的劇情事件特別有用。

每個預設事件包含：
- **名稱**和**描述**
- **可見性**：是否在觸發時讓玩家看到事件名稱
- **多個動作**（依序執行）：

| 動作類型 | 說明 | 目標 |
|----------|------|------|
| **廣播訊息** | 向角色推送劇情文字 | 全體或指定角色 |
| **數值調整** | 修改角色數值（可設為永久或時效性） | 全體或指定角色 |
| **揭露隱藏資訊** | 解鎖角色的指定隱藏資訊 | 指定角色 |
| **揭露隱藏任務** | 解鎖角色的指定隱藏目標 | 指定角色 |

預設事件可重複觸發。每次觸發後，GM 可以看到每個動作的執行結果摘要（成功/跳過/失敗及原因）。

> 預設事件在準備階段和遊戲進行中都可以新增或編輯，但只能在遊戲進行中觸發。觸發入口位於控制台的「事件選單」區塊（詳見[6.4 控制台](#64-遊戲進行中控制台)）。

## 6. 遊戲流程

### 6.1 遊戲生命週期

```
準備階段（設計角色、調整數值、提前發放 PIN 給玩家預覽）
    ↓  點擊「開始遊戲」
遊戲進行中（玩家可互動，所有操作即時同步）
    ↓  點擊「結束遊戲」
遊戲結束（僅可查看預覽模式，互動功能關閉）
```

### 6.2 Baseline 與 Runtime

這是 LARP Nexus 的核心機制：

- **Baseline（原始設定）**：你在準備階段設定的角色初始狀態。開始遊戲後不會被修改，永遠是「角色卡原本的樣子」
- **Runtime（即時狀態）**：開始遊戲時，系統自動從 Baseline 複製一份。遊戲中所有操作（用技能、消耗物品、被攻擊...）都反映在 Runtime 上

> 簡單來說：Baseline 是存檔，Runtime 是目前進度。

### 6.3 開始遊戲前的準備

1. 確認所有角色設定完成（數值、技能、物品、隱藏資訊）
2. **提前數天**將 **PIN 碼**和角色卡連結（或 QR Code）發給每位玩家，讓他們以預覽模式熟悉角色與世界觀
3. 可先分享世界觀頁面連結，讓玩家預習
4. 遊戲當天，點擊「開始遊戲」，再將**遊戲代碼**公布給玩家

### 6.4 遊戲進行中：控制台

遊戲開始後，劇本管理頁面會新增「控制台」分頁，提供即時遊戲管理功能。控制台包含以下區塊：

| 區塊 | 說明 |
|------|------|
| **角色狀態總覽** | 即時監控所有角色的數值，透過 WebSocket 自動更新。點擊角色卡片可查看該角色的完整數值 |
| **事件紀錄** | 遊戲中所有事件的歷史紀錄（廣播、數值變化、對抗結果等） |
| **事件選單** | 從下拉選單選擇預設事件，一鍵觸發多動作事件 |
| **廣播訊息** | 向全體或指定角色推送劇情事件，支援標題和內文 |

廣播支援兩種模式：
- **全體廣播**：向所有角色推送（適合全場劇情事件）
- **個別推送**：向指定角色推送（適合私人劇情線索）

> 在角色管理頁面中，GM 仍可調整角色數值、手動揭露隱藏資訊和任務。

### 6.5 結束遊戲

1. 點擊「結束遊戲」
2. 在確認對話框中可為快照命名（例如「第二輪結局」），留空則自動以時間戳命名
3. 系統保存所有角色的最終狀態（快照）
4. 玩家僅能以預覽模式查看角色卡，互動功能關閉
5. Baseline 保持不變，下次可重新開始遊戲

# 第二部分：玩家指南

## 7. 遊戲前的準備

### 7.1 你會收到什麼

在遊戲開始前（通常是提前幾天），GM 會給你：
- 一個 **QR Code** 或角色卡連結
- 一組 **PIN 碼**（4 位數字）— 部分角色可能不設 PIN，直接開啟連結即可查看

遊戲當天，GM 會再公布：
- 一組 **遊戲代碼**（6 位英數字，如 `A3KX9P`）

### 7.2 預覽你的角色

收到角色卡連結後，你就可以提前認識自己的角色了：

**有 PIN 碼的角色：**
1. 用手機掃描 QR Code（或直接開啟連結）
2. 在解鎖頁面只輸入 **PIN 碼**（不輸入遊戲代碼）
3. 進入**預覽模式**

**沒有 PIN 碼的角色：**
1. 開啟角色卡連結
2. 不需要輸入任何資訊，直接點擊「直接進入」按鈕
3. 進入**預覽模式**

預覽模式下：
- 你可以瀏覽角色的背景故事、數值、技能、物品等
- 畫面頂部會顯示「預覽模式」提示
- 所有互動功能禁用（這是正常的，遊戲還沒開始）
- 利用這段時間熟悉角色設定和世界觀

### 7.3 進入完整模式

遊戲當天，GM 公布遊戲代碼後：

**有 PIN 碼的角色：**
1. 在解鎖頁面同時輸入**遊戲代碼**和 **PIN 碼**
2. 進入**完整模式**——所有互動功能啟用

**沒有 PIN 碼的角色：**
1. 在解鎖頁面輸入**遊戲代碼**即可
2. 進入**完整模式**

畫面頂部會顯示「遊戲進行中」提示。

> 忘記遊戲代碼或 PIN？請聯繫你的 GM。

---

## 8. 認識你的角色卡

解鎖後，你會看到角色卡主畫面。上方是角色圖片和名稱的 Hero 區域，下方有五個分頁：

### 物品

你持有的物品清單。每個物品點開可以看到詳細資訊和可執行的操作（詳見[使用物品](#10-使用物品)）。

物品分為三種類型：
- **消耗品**：使用後扣減數量，數量歸零後無法再使用（但物品仍保留在清單中）
- **道具**：可重複使用，不扣減數量，但可能有使用上限和冷卻時間
- **裝備**：不能主動「使用」，只能穿戴/卸下。穿戴時提供常駐數值加成。已裝備的物品會以特殊標示顯示

### 技能

你可以使用的技能清單。每個技能點開可以看到效果說明和使用按鈕（詳見[使用技能](#9-使用技能)）。

### 資訊

資訊分頁內有三個子分頁：

| 子分頁 | 內容 |
|--------|------|
| **角色故事** | 你的背景故事，以區塊形式呈現（標題與段落） |
| **人物關係** | 你與其他角色的關係描述 |
| **額外資訊** | 隱藏資訊（已揭露的會顯示）、世界觀頁面連結 |

### 數值

你的角色能力值（如體力、魔力等），顯示為「當前值 / 最大值」。

數值分頁還會顯示兩種效果面板：
- **活躍效果**：被施加的臨時效果（例如持續 5 分鐘的力量增益），會顯示倒數計時，到期後數值自動恢復
- **裝備效果**：已穿戴裝備提供的常駐數值加成，卸下裝備後消失

### 任務

GM 為你設計的任務與目標：
- **一般任務**：一開始就可見的任務目標
- **隱藏目標**：一開始不會顯示，隨著遊戲推進（例如看到特定物品、揭露特定隱藏資訊）會自動出現，以特殊樣式標示提醒你這是新發現的任務
- 每個任務有狀態標示（待處理、進行中、已完成、失敗），狀態由 GM 或技能效果控制

## 9. 使用技能

1. 切換到「技能」分頁
2. 點擊想要使用的技能卡片
3. 查看技能效果說明

### 目標為自己的技能

點擊「使用技能」即可。依照技能的檢定方式，可能直接生效，也可能需要系統擲骰判定成敗（見下方檢定方式說明）。

### 需要選擇目標的技能

如果技能涉及對抗或需要指定對象：
1. 從下拉選單選擇目標角色
2. 點擊「使用技能」
3. 依據檢定方式，結果會有不同處理

### 四種檢定方式

| 檢定方式 | 運作方式 | 你需要做什麼 |
|----------|----------|-------------|
| **無檢定** | 直接生效，100% 成功 | 按下使用即可 |
| **隨機檢定** | 系統自動擲骰判定成敗 | 按下使用，等結果 |
| **對抗檢定** | 與目標比數值，高者贏 | 選目標後使用，等對方回應 |
| **隨機對抗** | 雙方各擲骰比大小 | 選目標後使用，等對方回應 |

### 偷取 / 移除物品

有些技能的效果是偷取或移除對方的物品：

1. 先選擇目標角色，正常使用技能
2. 如果涉及對抗，等待對抗結果
3. **成功後**，系統會跳出目標角色的物品列表
4. 選擇你要對其生效的物品，確認即完成
   - **偷取**：該物品會從對方移到你身上
   - **移除**：該物品會從對方身上消失

---

## 10. 使用物品

1. 切換到「物品」分頁
2. 點擊想要使用的物品卡片
3. 視物品類型和功能，你可以進行以下操作：

### 使用（消耗品/道具）

點擊「使用」，物品效果依檢定方式生效：
- **消耗品**：每次使用扣減數量，數量歸零後無法再使用（但物品仍保留在清單中）
- **道具**：不扣減數量，可反覆使用
- 如果物品設有**使用上限**，達到上限後無法再使用（卡片會顯示鎖定圖示）
- 如果物品設有**冷卻時間**，使用後需等待冷卻結束才能再次使用（卡片會變灰並顯示倒數）

如果物品有對抗效果（如偷取），流程與技能的對抗相同——選目標、等結果。

> 裝備類型的物品不能「使用」，只能穿戴/卸下。

### 展示

把物品「秀」給另一位角色看：
1. 點擊「展示」
2. 選擇你想展示給誰
3. 對方會收到通知，看到你展示的物品

> 展示不會消耗物品。某些隱藏資訊或隱藏任務的揭露條件是「看過特定物品」——展示物品可以觸發這些條件。

### 裝備（僅裝備類型）

裝備類型的物品可以穿戴和卸下：
1. 點擊裝備類型的物品卡片
2. 點擊「裝備」或「卸下」
3. 裝備後，數值加成立即生效；卸下後恢復原本數值

> 已裝備的物品在「數值」分頁會顯示對應的常駐效果面板。

### 轉移

把物品送給另一位角色：
1. 點擊「轉移物品」
2. 選擇接收對象
3. 物品從你的清單移到對方清單。如果對方已有同名物品，會合併數量

> 並非所有物品都能轉移，由 GM 設定。已裝備的物品在轉移時會自動卸下。

## 11. 對抗檢定

當你與另一位玩家發生對抗（不論是攻擊、偷竊或其他需要比拼的場景），系統會自動處理。

### 你是攻擊方

1. 使用帶有對抗效果的技能或物品，選擇目標
2. 系統通知對方
3. 等待對方回應（畫面會顯示等待中的提示）
4. 對方回應後，系統計算結果並通知雙方
5. 如果你贏了且效果是偷取/移除物品，接著選擇要對哪件物品生效

### 你是防守方

1. 收到對抗通知（螢幕會彈出對抗回應對話框）
2. 對話框會顯示：攻擊方是誰、使用了什麼技能/物品、你的基礎防禦值
3. 如果這場對抗允許使用物品或技能應對，你可以**擇一**選擇一件物品或一個技能來增強防禦（兩者互斥，不能同時使用；並非每場對抗都允許，取決於攻擊方技能的對抗設定）
4. 點擊「確認回應」
5. 系統計算結果並通知雙方

> 對抗有時間限制（3 分鐘）。超時未回應會以當前數值自動結算。

---

## 12. 通知系統

畫面右上角的通知圖示會顯示未讀通知數量。點擊可查看所有通知：

- **GM 推送的劇情事件**（廣播或個人訊息）
- **數值變化**（被攻擊、被治療、效果到期等）
- **對抗結果**（勝敗通知）
- **物品變動**（獲得或失去物品）
- **物品展示**（有人向你展示了物品）
- **隱藏資訊揭露**（新的隱藏資訊被解鎖）
- **任務更新**（新任務出現或狀態變更）

通知會在遊戲期間持續保留。

---

## 13. 離線與重新連線

LARP Nexus 使用 WebSocket 即時同步。如果你的網路暫時中斷：

- **重新整理頁面**後，系統會自動補送離線期間錯過的事件（數值變化、廣播等），你可以在通知面板中查看
- 未完成的對抗回應會自動恢復（在時限內），你可以繼續操作

> 遊戲中盡量保持網路連線。如果長時間斷線，重新整理頁面即可恢復最新狀態。

# 常見問題

## 通用

**Q: 需要安裝 App 嗎？**

不需要。LARP Nexus 是網頁應用，用手機瀏覽器直接開啟即可。建議使用 Chrome 或 Safari。

**Q: 多人可以同時查看同一張角色卡嗎？**

可以，但建議一人一張，避免操作衝突。

---

## GM 專用

**Q: 遊戲代碼在哪？**

建立劇本後自動生成，在劇本管理的「劇本資訊」分頁可以看到。

**Q: 可以在遊戲進行中修改角色設定嗎？**

可以。你可以隨時調整角色的 Runtime 數值、推送事件、揭露隱藏資訊。但 Baseline 在遊戲進行中不會改變。

**Q: 遊戲結束後還能重新開始嗎？**

遊戲結束會保存快照。如需再跑一次，可以重新開始遊戲來建立新的 Runtime。

**Q: AI 角色匯入支援哪些 AI 服務？**

支援 OpenAI、Google Gemini，以及任何 OpenAI 相容的 API（例如本地部署的 LLM）。在個人設定頁選擇「自訂」即可輸入自訂 Base URL。

**Q: 可以一次管理多場遊戲嗎？**

可以。每場遊戲有獨立的遊戲代碼和角色群，互不影響。但建議不要讓不同場次的遊戲使用相同的遊戲代碼，以免玩家混淆。

---

## 玩家專用

**Q: 忘記 PIN 碼怎麼辦？**

請聯繫 GM，GM 可以在後台查看。部分角色可能不設 PIN，直接開啟連結即可查看。

**Q: 忘記遊戲代碼怎麼辦？**

請聯繫 GM。遊戲代碼通常是全場共用的。

**Q: 預覽模式和完整模式差在哪？**

預覽模式是唯讀的，看到的是角色的原始設定（Baseline）。完整模式可以互動，看到的是遊戲進行中的即時狀態（Runtime）。遊戲開始前和遊戲結束後，都只能使用預覽模式。

**Q: 遊戲結束後還能看角色卡嗎？**

可以，但只能以預覽模式查看，互動功能已關閉。

**Q: 對抗時對方不回應怎麼辦？**

對抗有 3 分鐘時限。超時後系統會以對方當前數值自動結算。

**Q: 為什麼角色卡上突然多了一條隱藏資訊或新任務？**

這是自動揭露機制。你在遊戲中的某些行為（例如看到特定物品、獲得特定物品、其他隱藏資訊被揭露等）觸發了揭露條件，系統會自動為你解鎖對應的隱藏資訊或隱藏任務。
