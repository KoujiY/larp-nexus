# LARP Nexus 使用指南

歡迎使用 LARP Nexus！這是一套專為**實境角色扮演遊戲（LARP）**設計的數位角色卡管理系統。

傳統 LARP 使用紙本角色卡，GM 需要手動追蹤數值變化、道具流轉、技能使用。LARP Nexus 把這些通通數位化——GM 可以即時推送劇情、遠端調整角色狀態；玩家用手機就能查看角色卡、使用技能和道具，所有操作即時同步。

> **閱讀建議**：GM 請通讀全文；玩家可以直接跳到[第二部分：玩家指南](#part2)。

---

# 第一部分：GM（遊戲主持人）指南 {#part1}

## 1. 登入系統

LARP Nexus 使用 **Magic Link** 驗證，不需要密碼：

1. 前往首頁，點擊「GM Login」
2. 輸入你的 Email
3. 去信箱收取驗證信，點擊信中連結
4. 登入完成，自動進入 Dashboard

> 💡 Magic Link 有時效性，如果過期請重新申請。

---

## 2. 建立劇本

登入後，從 Dashboard 點擊「劇本管理」進入劇本列表。

### 新增劇本

1. 點擊「新增劇本」
2. 填寫劇本名稱與描述
3. 儲存後，系統會自動生成一組 **6 位英數字遊戲代碼**（如 `A3KX9P`）

> ⚠️ 建議每場遊戲使用不同的遊戲代碼。雖然系統允許重複，但不同劇本使用相同代碼可能導致玩家解鎖到錯誤的角色。

### 編輯劇本

劇本管理頁面有四個分頁：

| 分頁 | 功能 |
|------|------|
| **基本資訊** | 劇本名稱、描述、遊戲代碼顯示 |
| **世界觀** | 背景設定、前導故事、章節內容（可分階段公開） |
| **角色管理** | 建立與管理所有角色 |
| **廣播訊息** | 向全體或指定角色推送即時訊息 |

### 世界觀頁面

每個劇本有一個公開的世界觀頁面（路徑 `/g/[gameId]`），不需要 PIN 即可瀏覽。你可以把這個連結提前分享給玩家，讓他們在遊戲開始前熟悉世界觀。章節可以設定為隱藏，在適當時機才公開。

---

## 3. 設計角色

在劇本的「角色管理」分頁中，點擊「新增角色」。角色設定共有七個分頁：

### 3.1 基本資訊

- **名稱**、**描述**、**角色圖片**
- **PIN 碼**：4-6 位數字密碼，用來保護角色卡。開啟 PIN 鎖後，玩家必須輸入正確 PIN 才能查看

> 💡 建議提前幾天將 PIN 碼和角色卡連結發給玩家，讓他們用預覽模式提前熟悉自己的角色和世界觀。遊戲代碼則在遊戲當天再公布。

### 3.2 公開資訊

扮演該角色的玩家在遊戲開始前對自己角色的認知，包括：
- 背景故事
- 性格描述
- 人際關係（對象名稱 + 關係描述，可多筆）

> 注意：公開資訊是「對該角色公開」，不代表其他玩家也會看到。每張角色卡都是專屬的，背景中可能包含不希望其他玩家得知的內容。

### 3.3 秘密資訊

角色對自己不清楚的部分。這些資訊在遊戲開始時全部處於隱藏狀態，隨著遊戲推進可能會逐漸揭露：
- 每條秘密有**標題**和**內容**
- 可設定**自動揭露條件**，讓秘密在特定情境下自動解鎖（詳見 [3.8 自動揭露](#auto-reveal)）
- GM 也可以在遊戲進行中手動揭露

### 3.4 任務

GM 指派給角色的任務與目標：
- 每個任務有**標題**和**描述**
- 任務可以設為**隱藏目標**，搭配自動揭露條件，在適當時機才顯示給玩家
- 隱藏目標在揭露後會以特殊樣式標示，提醒玩家這是新發現的任務

### 3.5 數值

角色的能力值，例如體力、魔力、智力等：
- 每項數值有**名稱**、**當前值**、**最大值**
- 技能和道具的效果會修改這些數值

### 3.6 技能

角色可使用的能力。每個技能可以設定：

- **名稱**、**描述**、**圖片**
- **使用次數**：限制可使用幾次（-1 表示無限）
- **冷卻時間**：使用後需等待多久才能再次使用
- **效果**（可多個）：
  - **數值變化**：修改目標角色的數值（如扣血、補魔）
  - **偷取道具**：從目標角色偷走一件道具，移到自己身上
  - **移除道具**：從目標角色身上移除一件道具（道具會消失，不會轉移給使用者）
  - **自訂效果**：GM 自行定義的效果描述，由 GM 口頭裁定
- **檢定方式**（決定技能是否成功）：
  - **無檢定**：直接生效
  - **隨機檢定**：系統擲骰決定成敗
  - **對抗檢定**：與目標角色比較數值，高者獲勝
  - **隨機對抗**：雙方各擲一次骰，比大小決勝負
- **對抗設定**（僅對抗/隨機對抗時）：
  - 關聯數值（用哪項數值進行對抗）
  - 對手可使用的最大道具/技能數（設為 0 則防守方無法使用道具或技能應對）
  - 平手處理規則
- **標籤**：標籤會影響遊戲規則，目前支援兩種：
  - **戰鬥**：帶有此標籤的技能發起對抗時，防守方只能使用同樣帶有「戰鬥」標籤的道具或技能來應對
  - **隱匿**：帶有此標籤的技能在發動時，對方不會知道攻擊者是誰（通知中會隱藏使用者名稱）

### 3.7 道具

角色持有的物品。每個道具可以設定：

- **名稱**、**描述**、**圖片**
- **類型**：`消耗品`（使用次數歸零後無法再使用，但仍保留在道具清單中）或 `裝備`
- **數量**
- **效果**：與技能相同（數值變化、偷取道具、移除道具、自訂效果）
- **檢定方式**：與技能相同（無檢定、隨機、對抗、隨機對抗）
- **可轉移**：是否允許玩家主動將此道具轉移給其他角色
- **標籤**：與技能相同，帶有「戰鬥」標籤的道具可以用來回應戰鬥對抗，帶有「隱匿」標籤的道具在使用時會隱藏使用者身份

### 3.8 自動揭露系統 {#auto-reveal}

秘密資訊和隱藏任務可以設定**自動揭露條件**，讓系統在條件滿足時自動解鎖：

| 條件類型 | 觸發時機 | 範例 |
|----------|----------|------|
| 看過指定道具 | 透過展示或其他方式看到某些道具後觸發 | 「看過神秘信件後揭露真實身份」 |
| 獲得指定道具 | 透過偷取、轉移等方式取得某些道具後觸發 | 「拿到鑰匙後解鎖密室任務」 |
| 特定秘密已揭露 | 角色的其他秘密被揭露後連鎖觸發 | 「揭露真實身份後，連鎖揭露真正的目的」 |

條件可以設為**全部滿足**或**任一滿足**。

## 4. 開始與管理遊戲

### 4.1 遊戲生命週期

```
準備階段（設計角色、調整數值、提前發放 PIN 給玩家預覽）
    ↓  點擊「開始遊戲」
遊戲進行中（玩家可互動，所有操作即時同步）
    ↓  點擊「結束遊戲」
遊戲結束（僅可查看預覽模式，互動功能關閉）
```

### 4.2 Baseline 與 Runtime

這是 LARP Nexus 的核心機制：

- **Baseline（原始設定）**：你在準備階段設定的角色初始狀態。開始遊戲後不會被修改，永遠是「角色卡原本的樣子」
- **Runtime（即時狀態）**：開始遊戲時，系統自動從 Baseline 複製一份。遊戲中所有操作（用技能、消耗道具、被攻擊...）都反映在 Runtime 上

> 簡單來說：Baseline 是存檔，Runtime 是目前進度。

### 4.3 開始遊戲前的準備

1. 確認所有角色設定完成（數值、技能、道具、秘密資訊）
2. **提前數天**將 **PIN 碼**和角色卡連結（或 QR Code）發給每位玩家，讓他們以預覽模式熟悉角色與世界觀
3. 可先分享世界觀頁面連結，讓玩家預習
4. 遊戲當天，點擊「開始遊戲」，再將**遊戲代碼**公布給玩家

### 4.4 遊戲進行中的操作

| 操作 | 說明 |
|------|------|
| **廣播訊息** | 向全體或指定角色推送劇情事件。支援標題和內文 |
| **調整數值** | 直接修改任何角色的 Runtime 數值 |
| **揭露秘密** | 手動揭露角色的秘密資訊（不等自動揭露條件） |
| **更新任務** | 揭露隱藏任務 |
| **查看狀態** | 即時監控所有角色的數值、道具、生效中的效果 |

### 4.5 結束遊戲

1. 點擊「結束遊戲」
2. 系統自動保存所有角色的最終狀態（快照）
3. 玩家僅能以預覽模式查看角色卡，互動功能關閉
4. Baseline 保持不變，下次可重複使用

---

## 5. 實用工具

### 廣播訊息

廣播支援兩種模式：
- **全體廣播**：向所有角色推送（適合全場劇情事件）
- **個別推送**：向指定角色推送（適合私人劇情線索）

### QR Code

系統可以為劇本世界觀頁面和每張角色卡生成 QR Code，方便分享連結。

# 第二部分：玩家指南 {#part2}

## 6. 遊戲前的準備

### 6.1 你會收到什麼

在遊戲開始前（通常是提前幾天），GM 會給你：
- 一個 **QR Code** 或角色卡連結
- 一組 **PIN 碼**（4-6 位數字）

遊戲當天，GM 會再公布：
- 一組 **遊戲代碼**（6 位英數字，如 `A3KX9P`）

### 6.2 預覽你的角色

收到 PIN 碼後，你就可以提前認識自己的角色了：

1. 用手機掃描 QR Code（或直接開啟連結）
2. 在解鎖頁面只輸入 **PIN 碼**（不輸入遊戲代碼）
3. 進入**預覽模式**

預覽模式下：
- 你可以瀏覽角色的公開資訊、數值、技能、道具等
- 畫面頂部會顯示黃色「👁 預覽模式」提示
- 所有互動功能禁用（這是正常的，遊戲還沒開始）
- 利用這段時間熟悉角色設定和世界觀

### 6.3 進入完整模式

遊戲當天，GM 公布遊戲代碼後：

1. 在解鎖頁面同時輸入**遊戲代碼**和 **PIN 碼**
2. 進入**完整模式**——所有互動功能啟用
3. 畫面頂部會顯示綠色「🎮 遊戲進行中」提示

> 忘記遊戲代碼或 PIN？請聯繫你的 GM。

---

## 7. 認識你的角色卡

解鎖後，你會看到角色卡主畫面。上方是角色圖片和名稱，下方有五個分頁：

### 📋 資訊

- **公開資訊**：關於你的角色的已知資訊——背景故事、性格、人際關係。這些是其他角色也可以得知的內容
- **秘密資訊**：你的角色對自己不清楚的部分。遊戲開始時全部處於隱藏狀態，隨著遊戲推進會逐漸揭露——可能是自動觸發，也可能是 GM 手動揭露
- **世界觀連結**：點擊可查看遊戲的世界觀設定和章節

### 📊 數值

你的角色能力值（如體力、魔力等），顯示為「當前值 / 最大值」。

如果你身上有**時效性效果**（例如被施加了持續 5 分鐘的力量增益），會在這裡看到倒數計時。效果到期後數值會自動恢復。

### ✅ 任務

GM 為你設計的任務與目標：
- **一般任務**：一開始就可見的任務目標
- **隱藏目標**：一開始不會顯示，隨著遊戲推進（例如看到特定道具、揭露特定秘密）會自動出現，以特殊樣式標示提醒你這是新發現的任務

### 🎒 道具

你持有的道具清單。每個道具點開可以看到詳細資訊和可執行的操作（詳見下一節）。

### ⚡ 技能

你可以使用的技能清單。每個技能點開可以看到效果說明和使用按鈕（詳見下一節）。

---

## 8. 使用技能

1. 切換到「⚡ 技能」分頁
2. 點擊想要使用的技能卡片
3. 查看技能效果說明

### 目標為自己的技能

點擊「使用技能」即可。依照技能的檢定方式，可能直接生效，也可能需要系統擲骰判定成敗（見下方檢定方式說明）。

### 需要選擇目標的技能

如果技能涉及對抗或需要指定對象：
1. 從下拉選單選擇目標角色
2. 點擊「使用技能」
3. 依據檢定方式，結果會有不同處理

### 四種檢定方式

| 檢定方式 | 運作方式 | 你需要做什麼 |
|----------|----------|-------------|
| **無檢定** | 直接生效，100% 成功 | 按下使用即可 |
| **隨機檢定** | 系統自動擲骰判定成敗 | 按下使用，等結果 |
| **對抗檢定** | 與目標比數值，高者贏 | 選目標後使用，等對方回應 |
| **隨機對抗** | 雙方各擲骰比大小 | 選目標後使用，等對方回應 |

### 偷取 / 移除道具

有些技能的效果是偷取或移除對方的道具：

1. 先選擇目標角色，正常使用技能
2. 如果涉及對抗，等待對抗結果
3. **成功後**，系統會跳出目標角色的道具列表
4. 選擇你要對其生效的道具，確認即完成
   - **偷取**：該道具會從對方移到你身上
   - **移除**：該道具會從對方身上消失

---

## 9. 使用道具

1. 切換到「🎒 道具」分頁
2. 點擊想要使用的道具卡片
3. 視道具功能，你可以進行以下操作：

### 使用

點擊「使用」，道具效果依檢定方式生效。消耗品會扣減使用次數，用完後無法再使用（但道具仍會保留在清單中）。

如果道具有對抗效果（如偷取），流程與技能的對抗相同——選目標、等結果。

### 展示

把道具「秀」給另一位角色看：
1. 點擊「展示」
2. 選擇你想展示給誰
3. 對方會收到通知，看到你展示的道具

> 💡 展示不會消耗道具。某些秘密資訊或隱藏任務的揭露條件是「看過特定道具」——展示道具可以觸發這些條件。

### 轉移

把道具送給另一位角色：
1. 點擊「轉移道具」
2. 選擇接收對象
3. 道具從你的清單移到對方清單

> 並非所有道具都能轉移，由 GM 設定。

---

## 10. 對抗檢定

當你與另一位玩家發生對抗（不論是攻擊、偷竊或其他需要比拼的場景），系統會自動處理。

### 你是攻擊方

1. 使用帶有對抗效果的技能或道具，選擇目標
2. 系統通知對方
3. 等待對方回應（畫面會顯示等待中的提示）
4. 對方回應後，系統計算結果並通知雙方
5. 如果你贏了且效果是偷取/移除道具，接著選擇要對哪件道具生效

### 你是防守方

1. 收到對抗通知（螢幕會彈出對抗回應對話框）
2. 對話框會顯示：攻擊方是誰、使用了什麼技能/道具、你的基礎防禦值
3. 如果這場對抗允許使用道具或技能應對，你可以選擇使用來增強防禦（並非每場對抗都允許，取決於攻擊方技能的對抗設定）
4. 點擊「確認回應」
5. 系統計算結果並通知雙方

> ⏱ 對抗有時間限制（3 分鐘）。超時未回應會以當前數值自動結算。

---

## 11. 通知系統

畫面右上角的 🔔 圖示會顯示未讀通知數量。點擊可查看所有通知：

- **GM 推送的劇情事件**（廣播或個人訊息）
- **數值變化**（被攻擊、被治療、效果到期等）
- **對抗結果**（勝敗通知）
- **道具變動**（獲得或失去道具）
- **道具展示**（有人向你展示了道具）
- **秘密揭露**（新的秘密資訊被解鎖）
- **任務更新**（新任務出現或狀態變更）

通知會保留 24 小時，最多 50 則。

---

## 12. 離線與重新連線

LARP Nexus 使用 WebSocket 即時同步。如果你的網路暫時中斷：

- 重新連線後，系統會自動補送離線期間的事件
- 未完成的對抗回應會自動恢復，你可以繼續操作
- 如果數值在你離線期間被修改，重新連線時會彈出通知

> 💡 遊戲中盡量保持網路連線。如果長時間斷線，可以重新整理頁面來恢復狀態。

# 常見問題

## 通用

**Q: 需要安裝 App 嗎？**

不需要。LARP Nexus 是網頁應用，用手機瀏覽器直接開啟即可。建議使用 Chrome 或 Safari。

**Q: 多人可以同時查看同一張角色卡嗎？**

可以，但建議一人一張，避免操作衝突。

---

## GM 專用

**Q: 遊戲代碼在哪？**

建立劇本後自動生成，在劇本管理的「基本資訊」分頁可以看到。

**Q: 可以在遊戲進行中修改角色設定嗎？**

可以。你可以隨時調整角色的 Runtime 數值、推送事件、揭露秘密。但 Baseline 在遊戲進行中不會改變。

**Q: 遊戲結束後還能重新開始嗎？**

遊戲結束會保存快照。如需再跑一次，可以重新開始遊戲來建立新的 Runtime。

**Q: 可以一次管理多場遊戲嗎？**

可以。每場遊戲有獨立的遊戲代碼和角色群，互不影響。但建議不要讓不同場次的遊戲使用相同的遊戲代碼，以免玩家混淆。

## 玩家專用

**Q: 忘記 PIN 碼怎麼辦？**

請聯繫 GM，GM 可以在後台查看。

**Q: 忘記遊戲代碼怎麼辦？**

請聯繫 GM。遊戲代碼通常是全場共用的。

**Q: 預覽模式和完整模式差在哪？**

預覽模式是唯讀的，看到的是角色的原始設定（Baseline）。完整模式可以互動，看到的是遊戲進行中的即時狀態（Runtime）。遊戲開始前和遊戲結束後，都只能使用預覽模式。

**Q: 遊戲結束後還能看角色卡嗎？**

可以，但只能以預覽模式查看，互動功能已關閉。

**Q: 對抗時對方不回應怎麼辦？**

對抗有 3 分鐘時限。超時後系統會以對方當前數值自動結算。

**Q: 為什麼角色卡上突然多了一條秘密資訊或新任務？**

這是自動揭露機制。你在遊戲中的某些行為（例如看到特定道具、獲得特定物品、其他秘密被揭露等）觸發了揭露條件，系統會自動為你解鎖對應的秘密資訊或隱藏任務。
