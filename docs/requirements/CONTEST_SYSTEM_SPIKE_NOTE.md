## 1. 攻擊方使用一般數值變化技能或道具，防守方使用一般數值變化技能或道具回應
- 攻擊方判定成功，攻擊方收到 技能使用結果 的通知，防守方收到 受到影響 的通知，兩邊的通知都會記錄數值變化的細節
- 防守方判定成功，攻擊方收到 技能使用結果 和 受到影響 的通知，防守方收到 技能使用結果 的通知，攻擊方的 技能使用結果 只紀錄使用失敗，其他兩個通知一樣會記錄數值變化的細節

## 2. 攻擊方使用一般數值變化技能或道具，防守方不選擇技能回應
- 攻擊方判定成功，攻擊方收到 技能使用結果 的通知，防守方收到 受到影響 的通知
- 防守方判定成功，攻擊方收到 技能使用結果 的通知，只紀錄使用失敗，防守方不會收到任何通知

## 3. 攻擊方使用偷竊或移除道具的技能或道具，防守方使用一般數值變化技能或道具回應
- 攻擊方判定成功，攻擊方才能選擇防守方的道具，選擇後才進行結算，攻擊方收到 技能使用結果 的通知，防守方收到 受到影響 的通知，兩邊的通知都會記錄道具變動的細節
- 防守方判定成功，攻擊方收到 技能使用結果 和 受到影響 的通知，防守方收到 技能使用結果 的通知，攻擊方的 技能使用結果 只紀錄使用失敗，其他兩個通知一樣會記錄數值變化的細節

## 4. 攻擊方使用偷竊或移除道具的技能或道具，防守方使用偷竊或移除道具的技能或道具回應
- 攻擊方判定成功，攻擊方才能選擇防守方的道具，選擇後才進行結算，攻擊方收到 技能使用結果 的通知，防守方收到 受到影響 的通知，兩邊的通知都會記錄道具變動的細節
- 防守方判定成功，防守方接著選擇防守方的道具，選擇後才進行結算，攻擊方收到 技能使用結果 和 受到影響 的通知，防守方收到 技能使用結果 的通知，攻擊方的 技能使用結果 只紀錄使用失敗，其他兩個通知一樣會記錄道具變動的細節

## 5. 複合型的技能，例如狀態變化+偷竊，比照前面幾項辦理
- 攻擊方只要是判定失敗就一定收到 技能使用結果 的通知，顯示使用失敗
- 無論是攻擊方或防守方，只要技能使用成功，就會先收到 技能使用結果 的通知，紀錄狀態與道具變化
- 受到技能影響的一方會接著收到 受到影響 的通知，紀錄狀態與道具變化
- 偷竊或移除道具會多一個選擇道具的步驟，等這個結束後才會發送所有的通知

## 6. 重新整理的場合
- 攻擊方重新整理：會切回原本的技能或道具使用畫面，並呼叫原本的技能或道具 dialog，進入等待防守方回應的狀態
- 防守方重新整理：出現 對抗檢定 的 dialog (目前已這樣做)
- 選擇目標道具的場合重新整理：恢復重新整理前選擇道具的 dialog
- 這幾個 dialog 都無法主動關閉
